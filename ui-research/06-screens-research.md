## Дата: 2026-03-24

---

## 1. Экран урока с видео

### Паттерн 1: Split-screen — видео слева, материалы справа (Skyeng Vimbox)

- **Layout:** Экран делится примерно 50/50 или 60/40. Левая колонка — видео с учителем (WebRTC), правая — интерактивные упражнения/материалы урока. Видео с 2019 года стало крупнее, навигация по материалам улучшена.
- **Детали:** При наведении на видео учитель видит панель инструментов: оценки за слайды, качество соединения, трекер времени речи ученика, переключатель аудио/видео.
- **Мобайл:** На мобильном учитель/материал/словарные карточки доступны одновременно — адаптивный layout.
- **Пример:** Skyeng Vimbox (vimbox.skyeng.ru)
- **Применимость для Moodle BBB:** BigBlueButton открывается в отдельном окне/вкладке по умолчанию. Для split-screen эффекта — встроить BBB через iframe в кастомный шаблон страницы Moodle. Требует HTTPS + разрешение `allow="camera *;microphone *;display-capture *;"`.

### Паттерн 2: Видео на всю ширину сверху, материалы под ним (Foxford)

- **Layout:** Сверху — видео/трансляция вебинара (широкоформатное, 16:9), под видео — секция с теорией и заданиями. Навигация по урокам в левом боковом меню.
- **Детали:** Foxford использует вебинарный формат + записи. Студент видит: видеоплеер → теоретический материал → задания (ДЗ). Левое меню = дерево уроков курса.
- **Прогресс:** За выполнение заданий начисляются очки опыта (геймификация), отслеживается успеваемость.
- **Пример:** Foxford (foxford.ru)
- **Применимость для Moodle BBB:** Наиболее простая реализация. BBB-активность вверху страницы курса, ниже — материалы урока (файлы, задания). Не требует iframe-встраивания.

### Паттерн 3: Видео встроено в страницу занятия, рядом — слайды/тетрадь (Skysmart)

- **Layout:** Интерактивная рабочая тетрадь — основной экран. Видеосвязь с учителем — в углу (картинка-в-картинке или боковая панель). Упражнения интерактивны, автопроверка.
- **Детали:** Неверные ответы — красный, правильные — зелёный. Работает на всех устройствах. Учитель и ученик работают в одном документе синхронно.
- **Пример:** Skysmart (skysmart.ru / edu.skysmart.ru)
- **Применимость для Moodle BBB:** H5P или SCORM как интерактивные упражнения + BBB в отдельном окне для видеосвязи. BBB Breakout Rooms можно использовать для работы в парах.

### Ключевые параметры экрана урока (сводка)

| Элемент | Vimbox | Foxford | Skysmart |
|---------|--------|---------|---------|
| Видео | 50% слева | 100% сверху | Угол/боковая |
| Материалы | 50% справа | Под видео | Центр экрана |
| Чат | Встроен в BBB/видео | Чат вебинара | Панель справа |
| Навигация | Слайды урока | Левое меню | Список упражнений |
| Прогресс | Треккер речи учителя | Очки опыта | Цвета ответов |

## 2. Кабинет ученика (dashboard)

### Основной паттерн: "Что делать сегодня"

Все лидеры рынка показывают первым делом **ближайшее действие**, а не абстрактный прогресс:

1. **Расписание/следующий урок** — приоритет #1
2. **Незавершённые задания** — приоритет #2
3. **Прогресс по курсам** — приоритет #3

### Foxford — кабинет ученика

- **Дашборд содержит:**
  - Программы обучения (активные курсы)
  - План занятий + Календарь уроков
  - Прогресс и очки опыта (геймификация — за каждый уровень подарки)
  - Активные задания с дедлайнами
  - Раздел "Корзина" — активные заказы
- **Навигация:** Левое боковое меню раскрывается при наведении
- **Карточка курса:** Курс + кнопка перехода, статус занятий

### Skysmart — кабинет ученика

- **Вход:** student.skyeng.ru, авторизация через телефон/VK/Google/Яндекс
- **Дашборд содержит:**
  - Добавить предмет (онбординг)
  - Список заданий с дедлайнами
  - Прогресс выполнения заданий
  - Доп. ресурсы: онлайн-библиотека, вебинары, форум

### Умскул — кабинет ученика

- **Дашборд содержит:**
  - Активные курсы с расписанием вебинаров
  - Домашние задания с дедлайнами
  - Рабочие тетради
  - Уведомления о предстоящих событиях
- **Особенность:** Push-уведомления о предстоящих вебинарах и ДЗ

### Moodle — кабинет ученика (стандарт + плагины)

**Стандартный дашборд Moodle 4.5:**
- Timeline block — дедлайны по всем курсам
- Calendar block — расписание
- Recently accessed items block
- Course overview — карточки курсов с прогрессом

**Режимы отображения курсов:** Card / List / Summary

**Прогресс на карточке:** % выполнения активностей (требует Activity Completion включённым). Зелёная полоса прогресса на карточке курса.

**Рекомендуемые плагины:**
- `block_disealytics` (DiSEA Learner Dashboard) — визуальный прогресс студента
- `block_progress` (Progress Bar) — цветные полосы прогресса по всем курсам
- `block_dash` (Dash) — кастомные дашборды по ролям с Activity Completion widgets

### CSS-селекторы Moodle для кабинета ученика

```css
/* Карточка курса */
.dashboard-card { }
.coursebox .course-info-container { }

/* Прогресс на карточке */
.completion-progressbar { }
.completion-progressbar .bar { background-color: #28a745; }

/* Timeline блок */

/* Заголовок дашборда */

/* Блок "мои курсы" */
#page-my-courses .card-deck .card { }
.course-overview-card { }
```

## 3. Кабинет учителя

### Основной паттерн: "Действия требующие внимания сейчас"

1. **Непроверенные работы** — первый приоритет
2. **Расписание занятий** — когда следующий урок
3. **Список учеников** — кто отстаёт

### Foxford — кабинет учителя

- Добавление курсов повышения квалификации
- Создание своих классов/кружков
- Приглашение учеников (копировать ссылку → отправить)
- Проведение мини-классов (двусторонняя видеосвязь)
- Расписание составлено без пересечений

### Moodle — кабинет учителя (стандарт + плагины)

**Стандартные инструменты:**
- Gradebook — журнал оценок
- Assignment grading interface — проверка заданий
- Reports — отчёты об активности

**Рекомендуемые плагины для учителя:**
- `block_grade_me` (Grade Me) — блок непроверенных работ, агрегирует все незачтённые задания/квизы
- `block_dash` — dashboard с таблицей учеников, прогресс по курсу, фильтры
- Edwiser Rapid Grader — быстрая проверка нескольких работ в одном интерфейсе
- Configurable Reports — кастомные отчёты по активности студентов

### CSS-селекторы Moodle для кабинета учителя

```css
/* Страница задания — преподаватель */
.mod-assign-submissions-table { }
.gradingform_checklist { }

/* Журнал оценок */

.gradingform_rubric .criteria { }

/* BBB — кнопка "Начать урок" для учителя */
.mod_bigbluebuttonbn #join_button_input {
    background-color: #e63946;
    font-size: 1.2rem;
    padding: 12px 32px;
}

/* Список участников курса */
#page-user-index .userslist td.cell { }
```

## 4. Кабинет родителя

### Основной паттерн: "Мой ребёнок сегодня"

1. Посещаемость и пропуски
2. Незавершённые задания ребёнка
3. Оценки за последние занятия
4. Предстоящие уроки

### Foxford — кабинет родителя

- Мониторинг образовательного процесса
- Отслеживание достижений ребёнка
- Управление доступом
- Коммуникация с учителями

### Moodle — кабинет родителя (плагины)

**Специализированные плагины:**
- `auth_enrolmentor` (Autoenrol Parents) — автоматически назначает родителя как ментора к своему ребёнку по кастомному полю профиля. Даёт доступ к прогрессу ребёнка.
- `block_mis` (Parental Portal) — доступ к живым MIS-данным: посещаемость, оценки, расписание.
- Роль "Parent" в Moodle — встроенная возможность через `enrol_mentor` — родитель видит курсы и прогресс ребёнка в режиме чтения.

**Как настроить в Moodle 4.5:**
1. Site admin → Users → Accounts → Browse list of users
2. User profile → Mentors field (custom profile field)
3. Plugin `auth_enrolmentor` автоматически энролит родителя в те же курсы
4. Создать кастомную роль "Parent" с правами `moodle/course:view` без редактирования

### CSS для родительского дашборда

```css
/* Скрыть для роли родителя лишние блоки */
.role-parent .block_news_items { display: none; }
.role-parent .block_calendar_month { display: none; }

/* Показать только блок прогресса */
.role-parent .block_progress { display: block; }
```

## 5. BigBlueButton в Moodle — лучшие практики UI

### Встраивание BBB iframe

**Можно ли встроить iframe прямо в курс?**
Да, технически возможно, но требует:
- HTTPS на сервере Moodle
- BBB-сервер с включённым iframe-embedding (по умолчанию отключено, включается в настройках self-hosted или запросом у провайдера)
- iframe параметры:
```html
<iframe src="{join_url}"
        width="100%"
        height="700"
        allow="camera *;microphone *;display-capture *;"
        allowfullscreen>
</iframe>
```
- Пауза ~5 секунд после создания сессии перед показом iframe

**Рекомендация:** Для production использовать стандартный redirect (BBB открывается в новой вкладке) — стабильнее и нет проблем с браузерными политиками.

### Кнопка "Войти в класс" — максимально заметной

BBB-активность в Moodle 4.5 показывает кнопку Join. CSS для выделения:

```css
/* Большая заметная кнопка входа в класс */
.mod_bigbluebuttonbn .bbb-join-button,
.mod_bigbluebuttonbn #join_button_input {
    display: block;
    width: 100%;
    max-width: 400px;
    margin: 24px auto;
    padding: 16px 32px;
    font-size: 1.4rem;
    font-weight: 700;
    background-color: #e63946;
    color: #ffffff;
    border: none;
    border-radius: 8px;
    cursor: pointer;
    text-align: center;
    box-shadow: 0 4px 12px rgba(230, 57, 70, 0.4);
    transition: transform 0.2s, box-shadow 0.2s;
}

.mod_bigbluebuttonbn .bbb-join-button:hover,
.mod_bigbluebuttonbn #join_button_input:hover {
    transform: translateY(-2px);
    box-shadow: 0 6px 20px rgba(230, 57, 70, 0.5);
}

/* Статус комнаты */
.mod_bigbluebuttonbn .bbb-room-status {
    text-align: center;
    font-size: 1.1rem;
    margin-bottom: 16px;
    padding: 8px;
    border-radius: 4px;
}

/* Расписание сессий */
.mod_bigbluebuttonbn .bbb-room-type-room_recordings .recordings-table {
    margin-top: 24px;
}
```

### Расписание BBB-сессий для студента

Лучший подход — использовать **Calendar block** Moodle: BBB-активность автоматически добавляет событие в календарь Moodle если установить "Open" и "Close" даты в настройках активности.

Дополнительно — Timeline block на дашборде показывает предстоящие BBB-сессии как события с дедлайном.

```css
/* Выделить BBB-события в календаре особым цветом */
.calendar_event_module[data-module-type="bigbluebuttonbn"] {
    background-color: #e63946 !important;
    border-color: #c1121f !important;
}

/* Таймлайн — BBB событие */
.block_timeline .event-list-item[data-module-name="bigbluebuttonbn"] {
    border-left: 4px solid #e63946;
    padding-left: 12px;
}
```

### BBB Layout в комнате

BBB 2.6+ поддерживает кастомные layout через `customStyle` параметр в Join URL. Основные режимы:
- **Presentation** — основной слайд большой, видео сверху
- **Video** — видео на всю ширину, слайды скрыты
- **Custom** — через CSS переменные BBB клиента

## 6. Приоритетный список для реализации в Moodle

### Приоритет 1: Кнопка "Войти в класс" — КРИТИЧНО
**Что:** Большая красная кнопка на странице BBB-активности
**Реализация:** CSS в Raw SCSS темы, селектор `.mod_bigbluebuttonbn #join_button_input`
**Сложность:** Низкая — только CSS, 30 минут работы

### Приоритет 2: Следующий урок на дашборде ученика
**Что:** Timeline/Calendar блок показывает ближайшую BBB-сессию
**Реализация:** Включить Timeline block на дашборде + настроить даты в BBB-активностях
**Сложность:** Низкая — только настройка, 1 час

### Приоритет 3: Карточки курсов с прогрессом
**Что:** Зелёная полоса прогресса на карточке курса
**Реализация:** Включить Activity Completion для активностей + Course Completion в курсе
**CSS:** `.completion-progressbar .bar { background-color: #28a745; }`
**Сложность:** Средняя — нужно настроить completion для каждой активности, 2-3 часа

### Приоритет 4: Блок "Grade Me" для учителя
**Что:** Учитель сразу видит непроверенные работы
**Реализация:** Плагин `block_grade_me` — установить, добавить на дашборд учителя
**Сложность:** Низкая — установка плагина + настройка, 1 час

### Приоритет 5: Родительский доступ
**Что:** Родитель видит прогресс ребёнка
**Реализация:** Плагин `auth_enrolmentor` + настройка роли Parent
**Сложность:** Средняя — требует настройки профилей пользователей, 3-4 часа

### Приоритет 6: Кастомный дашборд по ролям (Dash плагин)
**Что:** Разные виджеты для учителя/ученика/родителя
**Реализация:** Плагин `block_dash` — создать шаблоны для каждой роли
**Сложность:** Высокая — требует изучения конфигуратора Dash, 1-2 дня

### Приоритет 7: Split-screen урок (видео + материалы)
**Что:** BBB видео + H5P/страница материалов рядом
**Реализация:** Кастомный шаблон Moodle activity (PHP/Mustache) + CSS grid
**Сложность:** Высокая — требует разработки кастомного шаблона, 2-3 дня

## 7. Итоговая архитектура экранов для нашей онлайн-школы

### Экран урока (рекомендация)
```
┌─────────────────────────────────────────────┐
│  [Название урока]          [Прогресс курса] │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────────────┐  ┌──────────────┐ │
│  │                     │  │  Материалы   │ │
│  │  [ВОЙТИ В КЛАСС]    │  │  урока       │ │
│  │  (красная кнопка)   │  │              │ │
│  │                     │  │  - Теория    │ │
│  │  Начало: 15:00      │  │  - Задание   │ │
│  │  Записи прошлых     │  │  - ДЗ        │ │
│  │  уроков             │  │              │ │
│  └─────────────────────┘  └──────────────┘ │
└─────────────────────────────────────────────┘
```
**CSS реализация — два столбца:**
```css
.mod_bigbluebuttonbn #region-main {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 24px;
}

@media (max-width: 768px) {
    .mod_bigbluebuttonbn #region-main {
        grid-template-columns: 1fr;
    }
}
```

### Дашборд ученика (рекомендация)
1. Hero-блок: "Следующий урок" (BBB событие из Calendar, выделено)
2. Карточки курсов с прогрессом (Activity Completion)
3. Timeline — предстоящие ДЗ и дедлайны
4. Блок прогресса (Progress Bar plugin)

### Дашборд учителя (рекомендация)
1. "Требуют проверки" (Grade Me block)
2. Расписание BBB-сессий на сегодня/неделю
3. Таблица учеников с прогрессом (Dash block)
4. Быстрые действия: создать задание, назначить ДЗ
